# Disclaimer

## Authorized Use Only

The agents in this repository are designed exclusively for **authorized security testing, research, and educational purposes**. This includes:

- Penetration testing with written authorization from the system owner
- Bug bounty programs within their defined scope
- Capture The Flag (CTF) competitions
- Personal lab environments you own and control
- Academic research and coursework
- Security awareness training

## Your Responsibility

By using these agents, you acknowledge that:

1. **You are solely responsible** for ensuring you have proper written authorization before testing any system
2. **Unauthorized access** to computer systems is illegal in most jurisdictions (CFAA in the US, Computer Misuse Act in the UK, Art. 323-1 du Code Pénal in France, etc.)
3. **These agents are tools**, not authorization — having a security tool does not grant permission to use it against any system
4. **The authors are not liable** for any misuse, damage, or legal consequences resulting from the use of these agents
5. **Scope boundaries must be respected** — never test systems outside your authorized scope, even if technically possible

## What These Agents Are NOT

- Not a substitute for professional security training
- Not a replacement for established penetration testing methodologies
- Not a guarantee of finding all vulnerabilities
- Not authorization to test any system
- Not legal advice

## Scope Guard

All Tier 2 (execution-capable) agents incorporate a scope guard that requires explicit scope declaration before executing any command. This is a safety measure, not a legal shield. The scope guard can be bypassed, so **the ultimate responsibility lies with you**.

## Reporting Vulnerabilities

If you discover vulnerabilities using these agents, follow responsible disclosure practices:

1. Report to the asset owner through their designated security contact
2. Allow reasonable time for remediation before any public disclosure
3. Do not access, modify, or exfiltrate data beyond what is necessary to demonstrate the vulnerability
4. Follow the vulnerability disclosure policy of the organization
